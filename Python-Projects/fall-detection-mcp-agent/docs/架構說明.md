## 二、角色與職責

### 1. AI Agent (agent_task.py / agent_cam.g5.py)

**職責**：
- 接收任務指令（MQTT 或直接執行）
- 呼叫 Camera MCP 取得影像
- 呼叫 LLM 進行視覺分析
- 根據分析結果決策下一步動作
- 呼叫 PTZ MCP 控制鏡頭轉動
- 透過 Monitor MCP 輸出日誌

**決策邏輯**（以 agent_cam.g5.py 為例）：
1. 取得畫面 → AI 分析 → 判斷是否有目標
2. 有目標：
   - 在左側 → 向右轉動鏡頭（追蹤）
   - 在右側 → 向左轉動鏡頭（追蹤）
   - 在中央 → 停止轉動，鎖定目標
3. 無目標 → 持續掃描（來回轉動）

### 2. Camera MCP (mcp_cam.g1.py / cam_task.py)

**職責**：
- 管理 USB 攝影機 / 影片檔案 / RTSP 串流
- 提供 get_frame() 工具回傳 Base64 影像
- 提供 MJPEG 串流供瀏覽器觀看

**傳輸方式**：SSE (Port 8808)

**工具列表**：
| 工具名稱 | 功能 | 回傳格式 |
|---------|------|---------|
| get_frame | 取得最新畫面 | data:image/jpeg;base64,{base64} |

### 3. PTZ MCP (sg90_mcp_a.6.py)

**職責**：
- 控制 ESP32 上的 SG90 伺服馬達
- 接收角度指令轉動鏡頭

**傳輸方式**：HTTP (Port 80)

**工具列表**：
| 工具名稱 | 功能 | 參數 |
|---------|------|------|
| set_servo_angle | 轉動到指定角度 | angle: 0~180 |

### 4. Monitor MCP (mcp_msg.g1.py)

**職責**：
- 接收 Agent 傳來的日誌訊息
- 透過 WebSocket 推送給前端網頁

**傳輸方式**：stdio (本地程序)

**工具列表**：
| 工具名稱 | 功能 | 參數 |
|---------|------|------|
| send_log_to_web | 發送日誌訊息 | message: string |

### 5. MQTT Broker (Mosquitto)

**職責**：
- 作為任務派發中心
- Agent 訂閱 agent/task 主題接收任務
- 外部可發布任務到 agent/task

## 三、通訊協定對照表

| 通訊方向 | 使用協定 | Port | 說明 |
|---------|---------|------|------|
| Agent → Camera MCP | SSE | 8808 | 呼叫 get_frame 工具 |
| Agent → PTZ MCP | HTTP | 80 | 呼叫 set_servo_angle |
| Agent → Monitor MCP | stdio | - | 發送日誌 |
| Camera MCP → Browser | HTTP | 8818 | MJPEG 串流 |
| Monitor MCP → Browser | WebSocket | 8817 | 即時日誌推送 |
| MQTT Client → Broker | MQTT | 1883 | 發布任務 |
| Broker → Agent | MQTT | 1883 | 訂閱接收任務 |

## 四、啟動順序（重要）

1. **LLM 伺服器**（llama.cpp）
2. **Camera MCP**（mcp_cam.g1.py）
3. **Monitor MCP**（mcp_msg.g1.py）
4. **PTZ MCP**（ESP32 自動啟動）
5. **MQTT Broker**（docker run）
6. **AI Agent**（agent_cam.g5.py）

> 注意：Camera MCP 必須在 Agent 之前啟動，否則 Agent 連線會失敗。
