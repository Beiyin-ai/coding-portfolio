# MCP Agent 系統部署步驟

## 一、環境準備

### 1.1 安裝 uv (Python 套件管理器)

\`\`\`bash
curl -LsSf https://astral.sh/uv/install.sh | sh
source ~/.bashrc
\`\`\`

### 1.2 建立專案虛擬環境

\`\`\`bash
cd ~
uv init v6
cd v6
\`\`\`

### 1.3 安裝 Python 套件

\`\`\`bash
uv add fastmcp openai opencv-python paho-mqtt aiohttp screeninfo
\`\`\`

## 二、啟動 LLM 伺服器

### 2.1 下載模型

\`\`\`bash
# 下載 Qwen3-VL-2B GGUF 模型
mkdir -p ~/model/unsloth
cd ~/model/unsloth

# 下載模型檔案（約 2GB）
wget https://huggingface.co/unsloth/Qwen3-VL-2B-Instruct-GGUF/resolve/main/Qwen3-VL-2B-Instruct-Q4_K_M.gguf
wget https://huggingface.co/unsloth/Qwen3-VL-2B-Instruct-GGUF/resolve/main/Qwen3-VL-2B-mmproj-F16.gguf
\`\`\`

### 2.2 啟動 llama-server

\`\`\`bash
llama-server \
  --model ~/model/unsloth/Qwen3-VL-2B-Instruct-Q4_K_M.gguf \
  --mmproj ~/model/unsloth/Qwen3-VL-2B-mmproj-F16.gguf \
  -ngl 99 \
  -c 8192 \
  --flash-attn on \
  --temp 0.9 \
  --host 0.0.0.0 \
  --port 8880
\`\`\`

## 三、啟動 MQTT Broker

### 3.1 使用 Docker 啟動 Mosquitto

\`\`\`bash
# 檢查 1883 埠是否已被佔用
nmap 172.17.0.1 -p1883 | grep "/tcp open" && exit 0

# 建立設定檔
cd
mkdir -p mosquitto_config
cat > ~/mosquitto_config/mosquitto.conf << "EOF"
allow_anonymous true
listener 1883 0.0.0.0
EOF

# 啟動 Docker 容器
docker run -d --restart unless-stopped -p 1883:1883 \
  -v ~/mosquitto_config/mosquitto.conf:/mosquitto/config/mosquitto.conf \
  -v ~/mosquitto_config/log:/mosquitto/log \
  eclipse-mosquitto:2.0.14
\`\`\`

## 四、啟動 MCP 伺服器

### 4.1 啟動 Camera MCP (SSE)

\`\`\`bash
cd ~/v6
uv run mcp_cam.g1.py
\`\`\`

> 確認攝影機正常：http://localhost:8818/mymcp_cam.mjpg

### 4.2 啟動 Monitor MCP (stdio)

\`\`\`bash
cd ~/v6
uv run mcp_msg.g1.py
\`\`\`

> 確認日誌頁面：http://localhost:8817

### 4.3 啟動 PTZ MCP (ESP32)

將 `sg90_mcp_a.6.py` 上傳至 ESP32，修改 WiFi 設定後執行。

\`\`\`python
# 修改 WiFi 設定
SSID = "你的WiFi名稱"
PASSWORD = "你的WiFi密碼"
\`\`\`

確認 ESP32 IP 後，測試連線：

\`\`\`bash
uv run client_http_mcp.py -a "http://{ESP32_IP}:80/mcp"
\`\`\`

## 五、啟動 AI Agent

### 5.1 啟動視覺追蹤 Agent (agent_cam.g5.py)

\`\`\`bash
cd ~/v6
uv run agent_cam.g5.py
\`\`\\`

### 5.2 開啟監控畫面

- Agent 視覺畫面：http://localhost:8828
- 即時日誌：http://localhost:8817
- 攝影機原始串流：http://localhost:8818/mymcp_cam.mjpg

## 六、測試跌倒偵測

### 6.1 透過 MQTT 發布任務（agent_task 版本）

\`\`\`bash
mosquitto_pub -h 172.17.0.1 -p 1883 -t "agent/task" -m "那個人跌倒了嗎？簡單回答"
mosquitto_pub -h 172.17.0.1 -p 1883 -t "agent/task" -m "跌倒的人有沒有躺平在跑步機上？"
\`\`\`

### 6.2 直接使用 MCP 客戶端測試

\`\`\`bash
# 列出 Camera MCP 工具
uv run client_http_mcp.py -a "http://127.0.0.1:8808/sse"

# 呼叫 get_frame 工具
uv run use_mcp_tool.py -a "http://127.0.0.1:8808/sse" -t "get_frame" -p "{}"

# 控制 ESP32 伺服馬達
uv run use_mcp_tool.py -a "http://{ESP32_IP}:80/mcp" -t "set_servo_angle" -p {angle: 90}
\`\`\`

## 七、常見問題排除

### 7.1 Camera MCP 連線失敗

\`\`\`bash
# 檢查攝影機是否被佔用
lsof /dev/video0

# 檢查 port 8808 是否監聽中
netstat -tlnp | grep 8808
\`\`\`

### 7.2 stdio MCP 無法啟動

確認 `mcp_msg.g1.py` 中沒有 `print()` 語句，stdio 模式會將 stdout 作為通訊通道。

### 7.3 ESP32 無法連線

\`\`\`bash
# 檢查 ESP32 IP
ping {ESP32_IP}

# 檢查 MCP 端點是否正常
curl http://{ESP32_IP}:80/mcp
\`\`\`

## 八、系統關閉順序

1. Ctrl+C 停止 AI Agent
2. Ctrl+C 停止 Camera MCP
3. Ctrl+C 停止 Monitor MCP
4. Ctrl+C 停止 llama-server
5. 停止 MQTT 容器：\`docker stop $(docker ps -q --filter ancestor=eclipse-mosquitto)\`
