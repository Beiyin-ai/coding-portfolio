# MCP 通訊協定比較

## HTTP vs SSE vs stdio 對照表

| 項目 | HTTP | SSE | stdio |
|------|------|-----|-------|
| **全名** | Hypertext Transfer Protocol | Server-Sent Events | Standard Input/Output |
| **連線方式** | 短連線，一次請求一次回覆 | 長連線，持續推送 | 程序間管道 |
| **資料方向** | 雙向（請求-回應） | 單向（伺服器→客戶端） | 雙向（stdin/stdout） |
| **是否需要重新請求** | 每次都要 | 不用 | 不用 |
| **適用場景** | 圖片、檔案、API | 即時訊息、通知 | 本地程序通訊 |
| **跨電腦** | ✅ 是 | ✅ 是 | ❌ 否 |
| **實現複雜度** | 低 | 中 | 低 |

## MCP 三種傳輸方式詳細說明

### 1. stdio（標準輸入輸出）

**適用場景**：
- 本地程序間通訊
- 同台電腦的 MCP server 與 client
- 不需要網路的純本地應用

**範例**：
\`\`\`python
# MCP Server (stdio)
mcp.run(transport="stdio")

# MCP Client
async with stdio_client(server_params) as (read, write):
    async with ClientSession(read, write) as session:
        await session.call_tool("tool_name", {})
\`\`\`

**注意事項**：
- 嚴禁在 server 中使用 print()，會污染通訊通道
- 需使用 sys.stderr 輸出除錯資訊

### 2. SSE (Server-Sent Events)

**適用場景**：
- 伺服器需要主動推送資料
- 即時日誌、狀態更新
- AI token 逐字輸出

**特點**：
- 基於 HTTP，可跨網路
- 單向推送，客戶端無法透過同一連線發送資料
- 自動重連機制

**範例**：
\`\`\`python
# MCP Server (SSE)
mcp.run(transport="sse", host="0.0.0.0", port=8808)

# MCP Client
async with sse_client("http://localhost:8808/sse") as (read, write):
    async with ClientSession(read, write) as session:
        await session.call_tool("tool_name", {})
\`\`\`

### 3. Streamable HTTP MCP

**適用場景**：
- 需要雙向串流通訊
- 大資料傳輸
- 跨網路的 AI 與工具互動

**特點**：
- 基於 HTTP
- 支援串流傳輸
- 雙向通訊

## 本專案中的使用

| 元件 | 傳輸方式 | 原因 |
|------|---------|------|
| Camera MCP | SSE | 需持續推送影像給 Agent |
| Monitor MCP | stdio | 本地日誌服務，無需網路 |
| PTZ MCP | HTTP | ESP32 資源有限，HTTP 較簡單 |
| Agent ↔ Camera | SSE | Agent 主動呼叫工具，不需伺服器推送 |
| Agent ↔ Monitor | stdio | 本地程序通訊 |
| Agent ↔ PTZ | HTTP | 跨網路控制 ESP32 |

## 選擇建議

| 需求 | 建議傳輸方式 |
|------|-------------|
| 同台電腦、本地工具 | stdio |
| 伺服器需主動推送 | SSE |
| 需要雙向串流、跨網路 | Streamable HTTP |
| 簡單的請求-回應 | HTTP |
